MySQL-DQL语言

数据库

基础指令

常见命令

--查看当前数据库或表：show databases/tables
--打开指定的库：use 库名
--创建表：
    create table 表名(
        列名 列类型
    )
--查看表结构：desc 表名

语法规范

--不区分大小写
--每条命令最好用分号结尾

排序

--语法：
    select 查询列表 from 表
        where 筛选条件
            order by 排序列表 asc/desc

--特点：
    1-asc代表的是升序，desc代表的是降序(不写默认升序)
    2-order by子句中可以支持单个字段、多个字段、表达式、函数、别名
    3-order by子句一般是放在查询语句的最后面，limit子句除外

分组查询：

--方法：group by语句
    <!-- 
        select 分组函数，列(要求出现在group by的后面)
            from 表
                [where 筛选条件]
                group by 分组的列表
                [order by 子句] 
        注意：
            查询列表必须特殊，要求是分函数和group by 后出现的字段
    -->

--特点：
    1-分组查询中的筛选条件分为两类
        （1）在分组前筛选的数据来自于原始表，语句置于group by前面，关键字是where
        （2）在分组后筛选的数据来自于分组后的结果集，语句置于group by 后面，关键字是having
            <!-- 能用分组前筛选的，优先考虑使用分组前筛选 -->
    2-group by子句支持单个字段分组，多个字段分组(多个字段之间用逗号隔开没有顺序要求)，表达式或函数(用的较少)
    3-可以添加排序（排序放在整个分组之后）

连接查询：

--说明：又称多表查询 
--分类：
    <!-- sql92语法 -->
    1-内连接：
        （1）等值连接：
            --多表连接的结果为夺标的交集部分
            --n表连接，至少需要n-1个连接条件
            --多表的顺序没有要求
            --一般需要为表起别名
        （2）非等值连接
        （3）自连接

    <!-- 
        sql99语法：
            格式：
                select 列 from 表1 别名 [连接类型]
                    join 表2 别名 on 连接条件
                        ........
        -->
    1-内连接（inner）:
        特点：
            （1）innner可以省略
            （2）筛选条件放在where后面，连接条件放在on后面，提高分离性，便于阅读
            （3）与sql192当中的等值连接相同
    2-外连接：
        （1）左外：left [outer]
        （2）右外：right [outer]
        （3）全外：full [outer]
        特点：
            --外连接的查询结果为主表中的所有记录（有则匹配，无则为null）
            --左外喝右外交换两个表的顺序，可以实现同样的效果
    3-交叉连接（cross）

子查询

--说明：出现在其他语句内部的select语句，称为子查询或内查询，内部嵌套其他select语句的查询，称为外查询或主查询
--分类：
    1-子查询出现的位置：
        （1）select后面：只能支持标量子查询

        （2）from后面：支持表子查询，将子查询的结果充当一张表，要求必须起别名

        （3）where或having后面：支持标量子查询、列子查询、行子查询
                特点：
                    --子查询放在小括号内
                    --子查询一般房子条件的右侧
                    --标量子查询，一般搭配着单行操作符使用（> < >= <= = <>）
                    --列子查询，一般搭配着多行操作符使用（in、any/some、all）
                    --子查询的执行优先于主查询

        （4）exists后面（相关子查询）：支持· 表子查询

    2-按结果集的行列数不同：
        （1）标量子查询（结果集只有一行一列）
        （2）列子查询（结果集只有一列多行）
        （3）行子查询（结果集只有一行多列）
        （4）表子查询（结果集一般为多行多列）

分页查询

--说明：当要显示的数据，一页显示不全，需要分页提交sql请求
--语法：
    select 查询列表 from 表
        【[join type] join 表2 on 连接条件
            where 筛选条件 group by 分组字段
                having 分组后的筛选
                    order by 排序字段】
        limit 【offset，】size;
            <!-- 
                offset：要显示条目的起始索引(起始索引从0开始)
                size：要显示条目的个数 -->
--特点：
    1-limit语句放在查询语句的最后

联合查询

--说明：union联合合并，将多条查询语句的结果合并成一个结果

--语法：
    查询语句1
        union
    查询语句2

--特点：
    1-要求多条查询语句的查询列数是一致的
    2-要求多条查询语句的查询的每一列的类型喝顺序最好一致
    3-union关键字默认去重，如果使用union all可以包含重复项

常见函数

--单行函数：
    a.字符函数：
        1-length：获取参数值的字节个数
        2-concat：拼接字符串
        3-upper、lower：将参数进行大小写变换
        4-substr：截取字符串
        5-instr（pos,find）：返回子串第一次出现的索引，如果找不到返回0
        6-trim（delete,pos）：去除前后一样的子串
        7-lpad、rpad（pos,length,fill）：用指定的字符实现左右填充指定长度
        8-replace（pos,find,replace）：替换
    b.数学函数:
        1-round：四舍五入
        2-ceil：向上取整
        3-floor：向下取整
        4-truncate：截断
        5-mod：取余 
    c.日期函数
        1-now：返回当前系统日期+时间
        2-curdate：返回房前系统日期，不包含时间
        3-curtime：返回当前时间，不包含日期
        4-可以获取指定部分的那年、月、日、小时、分钟、秒
        5-str_to_date：将日期格式的字符转换成日期
        6-date_format：将日期转换成字符
        7-datediff：日期相减
    d.其他函数：
        1-version：版本号
        2-database：当前数据库
        3-user：用户
    e.流程控制函数
        1-if（条件,trueResault,falseResault）：if else 效果
        2-case函数：
            （1）case 判断的字段或表达式
                    when 常量1 then 显示的值1或语句1
                    ...........
                    else 要显示的值n或语句n
                end 

            （2）case
                    when 条件1 then 要显示的值1或语句1
                    ................
                    else 要显示的值n或语句n
                end 

--分组函数：
    a.功能：用作统计使用，又称聚合函数/统计函数/组函数
    b.分类：
        1-sum：总和
        2-avg：平均值
        3-max/min：最大值和最小值
        4-count：数量
    c.注意：
        1-sum、avg只能处理数值类型数据
        2- 分组函数都忽略null值
        3-可以和distinct搭配实现去重
        4-和分组函数一同查询的字段要求是group by后的字段
