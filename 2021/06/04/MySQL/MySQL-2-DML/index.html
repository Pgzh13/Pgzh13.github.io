MySQL-DML语言

#数据库

##数据操作语言

###插入语句

--语法：
    a.方式1：
        insert into 表名(列名,........) valuss(值1,.......)
    b.方式2：
        insert into 表名
            set 列名=值,......

            <!-- 
                二者的区别：
                    1-方式1支持插入多行
                    2-方式1支持字符查询，方式2不支持
              -->

--特点：
    a.插入的值的类型要与列的类型一致或兼容
    b.不可以为null的列插入值
    c.列的顺序是否可以调换
    d.列数和值的个数必须一致
    e.可以省略列名，默认所有列，而且列的顺序和表中列的顺序一致

###修改语句

--语法：
    a.修改单表的记录
        update 表名 set 列=新值
            where 筛选条件;
    b.修改多表的记录
        <!-- sql92 -->
        update 表1 别名，表2 别名
            set 列=值
                where 连接条件
                    and 筛选条件；

        <!-- sql99 -->
        update 表1 别名 
            连接类型 join 表2 别名
                on 连接条件
                    set 列=值
                        where 筛选条件

###删除语句

--语法：
    a.方式1：delete语句
        1-单表删除
            delete from 表名 where 筛选条件
        2-多表删除
            <!-- sql92 -->
            delete 表1的别名，表2的别名
                from 表1 别名，表2 别名
                    where 连接条件 and 筛选条件

            <!-- sql99 -->
            delete 表的别名，表2的别名
                from 表1 别名
                    连接类型 join 表2 别名
                        on 连接条件
                            where 筛选条件

    b.方式2：truncate语句

        <!-- 
            二者的区别：    
                1-delete 可以加where条件，truncate不能加
                2-truncate删除，效率更高
                3-假如要删除的表钟有自增长列，如果用delete删除后，再插入数据，自增长列的断点开始，而truncate删除后插入数据，自增长列的值从1开始 
                4-truncate删除没有返回值，delete删除有返回值
                5-truncate删除不能回滚，delete可以
        -->
